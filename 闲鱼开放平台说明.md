# 闲鱼开放平台说明

## 目标

当前这套方案已经按“多账号 + 批次 + 任务明细”来搭。

它解决 3 件事：

1. 一个商品可以发到多个闲鱼账号
2. 你可以自己选择哪些商品发到哪个账号
3. 后续可以按账号或批次查询“哪些已上传、哪些失败、哪些还没上传”

后面要做网页可视化，这套数据结构可以直接复用。

## 当前数据结构

### `xianyu_accounts`

闲鱼账号表。

主要字段：

- `account_name`
- `app_key`
- `app_secret`
- `merchant_id`
- `user_name`
- `province`
- `city`
- `district`
- `item_biz_type`
- `sp_biz_type`
- `stuff_status`
- `channel_pv_json`
- `enabled`

### `xianyu_publish_batches`

批次表。

主要字段：

- `account_id`
- `batch_name`
- `status`
- `total_count`
- `success_count`
- `failed_count`

### `xianyu_publish_tasks`

任务明细表。

一个账号下一个商品对应一条任务。

主要字段：

- `account_id`
- `batch_id`
- `product_id`
- `status`
- `publish_status`
- `third_product_id`
- `err_code`
- `err_msg`
- `publish_payload_json`

## 首次初始化

执行：

```bash
python3 init_db.py
python3 scripts/setup_xianyu_open.py
```

会生成 4 个 Excel：

- `xianyu_category_mapping.xlsx`
- `xianyu_publish_defaults.xlsx`
- `xianyu_accounts.xlsx`
- `xianyu_batch_selection.xlsx`

## 四个配置文件

### `xianyu_category_mapping.xlsx`

作用：本地商品分类映射到开放平台类目。

字段：

- `source`
- `source_category`
- `item_biz_type`
- `sp_biz_type`
- `channel_cat_id`
- `channel_cat_name`
- `enabled`
- `note`

现在上传时会优先使用这一行里配置的：

- `item_biz_type`
- `sp_biz_type`
- `channel_cat_id`

如果这两列留空，才会回退到账号默认值。

### `xianyu_publish_defaults.xlsx`

作用：全局默认参数。

字段：

- `key_name`
- `key_value`
- `note`

常用项：

- `item_biz_type`
- `sp_biz_type`
- `stuff_status`
- `user_name`
- `province`
- `city`
- `district`
- `channel_pv_json`
- `callback_url`

### `xianyu_accounts.xlsx`

作用：维护多个闲鱼账号。

字段：

- `account_name`
- `app_key`
- `app_secret`
- `merchant_id`
- `user_name`
- `province`
- `city`
- `district`
- `item_biz_type`
- `sp_biz_type`
- `stuff_status`
- `channel_pv_json`
- `enabled`
- `note`

说明：

- 每个账号都可以有自己的密钥和发布参数
- 如果账号字段为空，会回退到全局默认值

### `xianyu_batch_selection.xlsx`

作用：用表格选品，决定“哪个账号上传哪些商品”。

字段：

- `batch_name`
- `account_name`
- `product_id`
- `branduid`
- `enabled`
- `note`

说明：

- `product_id` 和 `branduid` 二选一填即可
- `account_name` 必须能在 `xianyu_accounts.xlsx` 找到
- 同一个 `batch_name + account_name` 会组成一个上传批次

## 查类目

如果你还不知道 `channel_cat_id` 和 `channel_cat_name`，先查类目：

```bash
python3 scripts/query_xianyu_categories.py --item-biz-type 1 --sp-biz-type 0
```

脚本会导出：

- `xianyu_categories_result.xlsx`

然后把结果填回 `xianyu_category_mapping.xlsx`。

## 同步配置到数据库

改完 Excel 后执行：

```bash
python3 scripts/sync_xianyu_open_config.py
```

## 回调与状态同步

上架接口是异步的，所以系统现在有 4 个关键状态：

- `created`: 第三方平台商品已创建
- `submitted`: 已提交上架请求
- `published`: 已通过回调确认上架成功
- `publish_failed`: 已通过回调确认上架失败

启动本地回调服务：

```bash
python3 scripts/run_xianyu_callback_server.py --host 0.0.0.0 --port 8787
```

回调地址示例：

```text
http://你的服务器地址:8787/xianyu/callback
```

然后把这个地址填到：

- `xianyu_publish_defaults.xlsx` 里的 `callback_url`

再同步配置：

```bash
python3 scripts/sync_xianyu_open_config.py
```

这样后续调用 `publish` 时会自动带上 `notify_url`。

如果当前还没有公网回调地址，但你已经肉眼确认商品已经上架，可以先手动回写：

```bash
python3 scripts/mark_xianyu_tasks_published.py --batch-id 11
```

这会把该批次任务从 `submitted` 标为 `published`。

## 生成批次任务

按选品表建批次和任务：

```bash
python3 scripts/create_xianyu_batches_from_selection.py
```

## 预览单任务 payload

```bash
python3 scripts/publish_xianyu_open_single.py --task-id 1
```

这一步不会调接口，只会生成 payload 并回写任务状态。

## 实际执行单任务

```bash
python3 scripts/publish_xianyu_open_single.py --task-id 1 --execute
```

如果只创建商品，不调用上架：

```bash
python3 scripts/publish_xianyu_open_single.py --task-id 1 --execute --skip-publish
```

## 执行整批任务

先预览一整个批次：

```bash
python3 scripts/publish_xianyu_batch.py --batch-id 1
```

真实执行整批：

```bash
python3 scripts/publish_xianyu_batch.py --batch-id 1 --execute
```

只创建不调用上架：

```bash
python3 scripts/publish_xianyu_batch.py --batch-id 1 --execute --skip-publish
```

如果只想先跑前 5 个：

```bash
python3 scripts/publish_xianyu_batch.py --batch-id 1 --execute --limit 5
```

## 导出账号任务结果

导出所有账号任务：

```bash
python3 scripts/export_xianyu_account_tasks.py
```

只导出某个账号：

```bash
python3 scripts/export_xianyu_account_tasks.py --account-name "闲鱼账号1"
```

如果还想一起导出该账号“未上传”的商品：

```bash
python3 scripts/export_xianyu_account_tasks.py --account-name "闲鱼账号1" --include-not-uploaded
```

导出结果现在包含 3-4 个 sheet：

- `account_summary`
- `batches`
- `tasks`
- `not_uploaded`（仅指定账号时可选）

## 本地管理后台

启动后台：

```bash
python3 scripts/run_xianyu_admin_server.py --host 127.0.0.1 --port 8790
```

打开：

```text
http://127.0.0.1:8790
```

当前可用功能：

- 查看账号汇总
- 查看批次
- 查看任务
- 查看回调记录
- 在账号页按分类筛选账号商品池
- 分类以按钮形式切换
- 在账号页分页浏览商品
- 在账号页查看商品图
- 在账号页看到“已上传/未上传”标记、最近状态、上传次数
- 在账号页勾选商品直接生成批次
- 在批次页直接点击“执行创建”“执行上架”
- 在账号任务列表里可手动点击“下架”

这就是后续员工网页登录版的基础形态，后面只需要再加登录和权限。

## 重复上传规则

系统现在允许：

- 同一个账号
- 对同一个商品
- 重复生成多个上传任务

后台商品池不会再把“已上传商品”隐藏掉，而是会显示：

- 是否已上传
- 最近状态
- 上传次数

所以员工登录后台后，看到的会是：

- 账号 A 下已上传过的商品会显示“已上传”
- 账号 B 下没上传过同一商品，则会显示“未上传”

这两个账号彼此独立。

## 图片稳定性

当前上传图片默认优先使用原站公网图片 URL。

如果后面你有自己的 OSS/CDN，可以配置：

```bash
export XIANYU_IMAGE_CDN_BASE_URL="https://你的图片域名"
```

配置后，系统会尝试把本地 `data/` 下的图片路径映射成这个域名下的外链，优先用于第三方上传。

这一步的作用是：

- 不再依赖原站图片长期可访问
- 提升闲鱼列表缩略图稳定性

## 无库存自动下架

系统现在已经支持：

- 每天库存更新后
- 如果某个已上传/已上架商品库存为 0
- 或本地商品状态已经变成 `inactive`
- 会自动调用闲鱼开放平台的下架接口

对应脚本：

```bash
python3 scripts/auto_downshelf_zero_stock.py
```

这个脚本已经接入：

```bash
python3 daily_sync.py
```

所以你以后日常跑 `daily_sync.py` 时，会自动完成这一步。

下架后的任务状态会写成：

- `status = off_shelved`
- `publish_status = off_shelved`

如果下架失败，会写成：

- `status = off_shelf_failed`
- `publish_status = off_shelf_failed`

## 当前 payload 规则

- 发布价优先取 `xianyu_publish_tasks.publish_price`
- 如果没填，就取 `product_updates.final_price_cny`
- 原价取 `product_updates.original_price_cny`
- 库存会从 `product_updates.stock` 尽量汇总为总库存
- 图片优先级：
  - `cover_image_path`
  - `local_image_path`
  - `image_url`
- 账号参数优先级：
  - 任务级 `channel_pv_json`
  - 账号级配置
  - 全局默认配置

## 当前限制

当前版本仍有这些限制：

1. 还没做回调接收服务
2. 还没做真正的网页后台
3. 还没做多规格 SKU 发布
4. 图片字段是否需要先上传成平台资源，联调时还要确认

## 以后做网页是否可行

可行，而且现在的数据结构就是按网页化准备的。

后面网页版可以直接做成：

1. 员工登录系统
2. 选择自己的闲鱼账号
3. 浏览商品列表
4. 勾选要上传的商品
5. 生成批次
6. 发起上传
7. 查看自己账号下的上传结果

也就是说，后面网页只是把现在这套 Excel 操作换成界面操作，不需要推倒重来。
