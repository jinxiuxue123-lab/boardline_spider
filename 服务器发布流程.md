# 服务器发布流程

## 目标

- 本地：开发环境
- 服务器：稳定环境

不要在服务器上直接改业务代码。所有新功能先在本地开发、测试，再发布到服务器。

## 本地开发

本地继续用现有方式运行：

```bash
./scripts/run_admin_server.sh
```

需要改代码时，只改本地项目。

## 推荐流程

现在项目已经接入 GitHub：

```bash
https://github.com/jinxiuxue123-lab/boardline_spider
```

推荐把发布拆成两段：

- 本地：开发、测试、提交、推送
- 服务器：拉最新稳定版并重启服务

## 本地发布步骤

```bash
cd /Users/jinxiuxue/Desktop/boardline_spider
git status
git add 需要提交的文件
git commit -m "说明这次改动"
git push
```

## 服务器发布步骤

服务器里执行：

```bash
cd /root/boardline_spider
bash scripts/deploy_server_update.sh
```

这个脚本会自动执行：

- `git pull --ff-only`
- 同步 `systemd` 配置
- `systemctl daemon-reload`
- `systemctl restart boardline-admin`
- 本机健康检查 `curl http://127.0.0.1:8790/`

## 手动发布到服务器

如果你想拆开执行，等价命令是：

```bash
cd /root/boardline_spider
git pull --ff-only
cp /root/boardline_spider/deploy/systemd/boardline-admin.service /etc/systemd/system/
cp /root/boardline_spider/deploy/systemd/boardline-daily.service /etc/systemd/system/
cp /root/boardline_spider/deploy/systemd/boardline-daily.timer /etc/systemd/system/
cp /root/boardline_spider/deploy/systemd/boardline-one8-daily.service /etc/systemd/system/
cp /root/boardline_spider/deploy/systemd/boardline-one8-daily.timer /etc/systemd/system/
systemctl daemon-reload
systemctl restart boardline-admin
systemctl status boardline-admin --no-pager -l
curl -i http://127.0.0.1:8790/ | head -20
```

## 旧的部署包方式

如果 GitHub 暂时不可用，仍然可以回退到部署包方案。

### 1. 本地生成部署包

```bash
./scripts/create_deploy_bundle.sh
```

默认会生成：

```bash
boardline_spider_deploy.tar.gz
```

这个部署包只包含代码和脚本，不包含：

- `data/`
- `logs/`
- 本地数据库
- `.git/`
- 虚拟环境

### 2. 上传部署包

在本机执行：

```bash
scp /Users/jinxiuxue/Desktop/boardline_spider/boardline_spider_deploy.tar.gz root@47.80.63.228:/root/
```

### 3. 服务器解压覆盖代码

在服务器执行：

```bash
cd /root
tar -xzf boardline_spider_deploy.tar.gz
```

如果服务器已经有稳定运行中的数据库和图片目录，不要删除：

- `/root/boardline_spider/products.db`
- `/root/boardline_spider/data/`

### 4. 重启后台

```bash
systemctl restart boardline-admin
systemctl status boardline-admin --no-pager -l
```

## 首次安装 systemd

把模板复制到系统目录：

```bash
cp /root/boardline_spider/deploy/systemd/boardline-admin.service /etc/systemd/system/
cp /root/boardline_spider/deploy/systemd/boardline-daily.service /etc/systemd/system/
cp /root/boardline_spider/deploy/systemd/boardline-daily.timer /etc/systemd/system/
cp /root/boardline_spider/deploy/systemd/boardline-one8-daily.service /etc/systemd/system/
cp /root/boardline_spider/deploy/systemd/boardline-one8-daily.timer /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now boardline-admin
systemctl enable --now boardline-daily.timer
systemctl enable --now boardline-one8-daily.timer
```

## 数据同步原则

代码发布和数据同步分开处理。

### 代码发布

只传部署包。

### 数据同步

需要时单独同步：

- `products.db`
- `data/images/`
- `data/ai_generated/`

例如：

```bash
scp /Users/jinxiuxue/Desktop/boardline_spider/products.db root@47.80.63.228:/root/boardline_spider/
scp -r /Users/jinxiuxue/Desktop/boardline_spider/data/images/boardline root@47.80.63.228:/root/boardline_spider/data/images/
scp -r /Users/jinxiuxue/Desktop/boardline_spider/data/images/one8 root@47.80.63.228:/root/boardline_spider/data/images/
scp -r /Users/jinxiuxue/Desktop/boardline_spider/data/ai_generated root@47.80.63.228:/root/boardline_spider/data/
```

同步数据库前，建议先备份服务器版本：

```bash
cd /root/boardline_spider
cp products.db products.db.bak_$(date +%Y%m%d_%H%M%S)
```

## 建议

- 服务器只跑稳定版。
- 本地继续开发和测试。
- 每次发布前，先在本地验证。
- 服务器不要直接改业务代码。
- 现在服务器稳定版默认跟踪：
  - `main`
- 如果后面要做多人协作，再补：
  - `dev`
  - 功能分支
